# Translation guide

English (`en_us`) is generated by datagen and is the canonical key set. Turkish (`tr_tr`) is
maintained in `src/main/resources/assets/projectex/lang/tr_tr.json`. Both ship with exact key
parity; an automated test fails for missing or extra keys.

To add a locale, copy generated `src/main/generated/assets/projectex/lang/en_us.json` to
`src/main/resources/assets/projectex/lang/<locale>.json`, translate values only, preserve
UTF-8 and every `%s`, `%1$s`, numeric placeholder, punctuation role, and JSON key. Do not
translate identifiers, commands, config keys, file paths, or EMC units. Keep labels suitable
for narration and narrow screens; never communicate state by color alone.

Run `./gradlew runDatagen` after English source changes, update every maintained locale, then
run `./gradlew check`. Review in-game with the locale selected, keyboard-only navigation,
narration enabled, long item names, errors, confirmations, and paging. A translation PR should
name the locale and reviewer fluency, change no gameplay behavior, and update this guide when
a maintained locale is added. Community translations follow the repository license,
contribution process, and code of conduct.
