# Security policy

## Supported versions

Until the first stable release, only the latest tagged pre-release receives security
fixes. After 1.0, supported Minecraft lines are listed in `docs/support-policy.md`.

## Report a vulnerability

Do not open a public issue for duplication exploits, arbitrary file/network access,
permission bypasses, denial of service, malicious packets/data packs, or sensitive
information exposure. Use GitHub's **Report a vulnerability** private advisory form
on the Security tab. If private reporting is not enabled yet, contact the repository
owner privately through their GitHub profile and disclose only that you need a secure
channel—do not include exploit details in public.

Include affected versions, environment, impact, minimal reproduction, and any
suggested mitigation. Expect acknowledgement within 7 days and a status update within
14 days. Timelines depend on severity and coordinated disclosure needs.

## Disclosure

Maintainers will validate, assign severity, prepare fixes privately, request a CVE
when appropriate, and coordinate release notes. Please allow a reasonable remediation
window before public disclosure. Good-faith research that avoids privacy violations,
data destruction, service disruption, and unauthorized access is welcomed.
